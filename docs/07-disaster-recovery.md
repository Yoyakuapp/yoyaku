# 07 Disaster Recovery

## 障害復旧

Yoyakuでは、予約・決済・管理画面の停止が店舗運営に影響します。障害時は影響範囲を素早く切り分け、予約データの整合性を最優先に復旧します。

## DB障害

DB障害時の確認項目:

- Neonのステータス
- 接続文字列の有効性
- Migration状態
- 接続数上限
- 直近のスキーマ変更

対応:

1. Neonダッシュボードで障害状況を確認
2. アプリケーションログでDB接続エラーを確認
3. 必要に応じて復旧ポイントを選定
4. 復旧後に予約データの整合性を確認

## GitHub障害

GitHub障害時は、新規デプロイやPull Request運用が影響を受けます。本番稼働中のアプリには通常影響しません。

対応:

- GitHub Statusを確認
- 緊急修正が必要な場合は復旧まで待機
- 復旧後に未反映の変更を確認

## Stripe障害

Stripe障害時は決済とWebhookに影響します。

対応:

1. Stripe Statusを確認
2. 新規予約受付への影響を確認
3. PaymentIntentの状態を確認
4. Webhook再送を確認
5. 決済成功・予約未確定がないか照合

## 復旧手順

1. 障害の種類を分類
2. 影響範囲を確認
3. 顧客・店舗への影響を記録
4. 復旧作業を実施
5. データ整合性を確認
6. 再発防止策をIssue化

## バックアップからの復元

バックアップ復元時は、復元ポイント以降の予約・決済との差分に注意します。Stripe側の決済履歴とDB上の予約状態を照合し、二重予約や未確定予約がないことを確認します。
