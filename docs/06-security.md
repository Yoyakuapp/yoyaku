# 06 Security

## セキュリティ運用

Yoyakuは予約情報、個人情報、決済情報を扱うため、セキュリティを事業運営の中心に置きます。

## 認証

管理画面はNextAuthによるメール・パスワード認証で保護します。パスワードはbcryptでハッシュ化し、平文では保存しません。

## 権限

初期段階では管理者権限を最小限の人数に限定します。多店舗化以降は、店舗単位・役割単位の権限管理を追加します。

## 秘密情報

秘密情報は環境変数で管理します。

- `.env` はコミットしない
- GitHubに秘密鍵を置かない
- VercelのEnvironment Variablesで本番値を管理
- Stripe secret keyとWebhook secretは閲覧権限を制限

## Stripe

カード情報はYoyakuのDBに保存しません。Stripe PaymentIntentとStripe.jsを使用し、決済成功Webhookを受けて予約を確定します。

## バックアップ

バックアップデータには個人情報が含まれるため、アクセス権限を制限します。復元作業時も必要最小限のメンバーで実施します。

## 個人情報

保存する個人情報は予約運営に必要な範囲に限定します。

- 氏名
- 電話番号
- メールアドレス
- 予約メモ

不要になった個人情報の削除方針は、正式公開前に定義します。

## ログ管理

ログには個人情報、カード情報、秘密情報を出力しません。障害調査に必要な情報は、識別子やステータスを中心に記録します。
