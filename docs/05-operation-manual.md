# 05 Operation Manual

## 運営マニュアル

Yoyakuの運営では、予約・決済・障害・セキュリティを継続的に確認します。

## 毎日

- 本日の予約状況を確認
- 決済失敗やWebhook失敗がないか確認
- 管理画面の主要機能が動作するか確認
- 顧客問い合わせを確認
- エラーログを確認

## 毎週

- 予約件数、キャンセル件数、Stripe入金・返金状況を確認
- DB容量とパフォーマンスを確認
- Stripe入金・返金状況を確認
- 未対応Issueを整理
- 依存パッケージの更新有無を確認

## 毎月

- 運営コストを確認
- セキュリティ設定を見直す
- バックアップと復旧手順を確認
- ロードマップを更新
- 顧客フィードバックを整理

## 障害対応

障害発生時は以下の順で対応します。

1. 影響範囲を確認
2. 予約・決済への影響を確認
3. 一時回避策を決定
4. 復旧作業を実施
5. 原因と再発防止策を記録

## 更新

更新作業はPull Requestを通して行います。本番反映前にbuild、test、lint、動作確認を行います。

## Git運用

- mainブランチは常にデプロイ可能な状態に保つ
- 作業はfeatureブランチで行う
- 変更内容は小さく分ける
- コミットメッセージは目的がわかる内容にする
- 秘密情報は絶対にコミットしない
