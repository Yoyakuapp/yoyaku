# 09 Roadmap

## 開発ロードマップ

Yoyakusは、マッサージ・リラクゼーション店舗向けの1店舗向け完成製品であるYoyakus Singleから始め、Group、Network、Platformへ段階的に拡張します。

## Stage 1: Yoyakus Single

目的: 1店舗向けの完成製品として、予約受付、予約金決済、店舗管理を運用できる状態にする。

- 顧客向け予約フロー
- 空き時間検索
- 予約金15%のStripe決済
- 決済成功時のみ予約確定
- 管理画面ログイン
- 予約一覧・予約詳細
- スタッフ・営業時間・休業日・シフト管理
- Organization 1件、Store 1件による店舗単位データ分離

### Single完成に必要な運用機能

- メール通知の外部サービス設定
- 顧客向けキャンセル導線
- 店舗情報管理のDB更新画面
- 監視・エラーログ
- 操作ログ

### Singleで完了した主要機能

- 管理者作成CLI
- 管理画面認証
- StoreMemberによる店舗スコープ
- スタッフ、営業時間、休業日、シフト管理
- メニュー管理のDB化
- 予約金決済とWebhookによる予約確定
- 管理者キャンセル、返金判定、施術完了
- 顧客管理と売上確認

## Stage 2: Yoyakus Group

目的: 同一事業者が複数店舗を管理できる状態にする。

- 複数Store
- 店舗別スタッフ・予約・決済
- 店舗別権限
- 店舗別売上
- 本部管理画面

## Stage 3: Yoyakus Network

目的: 複数事業者・複数店舗による地域の予約流通・送客ネットワークを作る。

- 地域・現在地検索
- 複数店舗の空き状況検索
- 店舗・時間・価格の比較
- サービス内容による絞り込み
- 近隣参加店舗の提案
- 送客実績管理
- 参加店舗ネットワーク

## Stage 4: Yoyakus Platform

目的: Single、Group、Networkを支える店舗運営共通基盤を整備する。

- テナント管理
- サブスクリプション課金
- オンボーディング
- 監査ログ
- SLA
- 分析ダッシュボード
