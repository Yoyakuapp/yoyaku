# 09 Roadmap

## 開発ロードマップ

Yoyakuは段階的にMVPからSaaSへ拡張します。

## Phase 1 MVP

目的: 単店舗で予約受付と予約金決済を運用できる状態にする。

- 顧客向け予約フロー
- 空き時間検索
- 予約金15%のStripe決済
- 決済成功時のみ予約確定
- 管理画面ログイン
- 予約一覧・予約詳細
- スタッフ・営業時間・休業日・シフト管理

## Phase 2 β

目的: 実店舗でβ運用し、運用上の不足を補う。

- メール通知
- 顧客向けキャンセル導線
- 返金運用の管理画面整備
- 顧客管理
- 店舗情報管理
- メニュー管理のDB化
- 監視・エラーログ
- 操作ログ

## Phase 3 正式公開

目的: 外部顧客に公開できる品質へ引き上げる。

- セキュリティレビュー
- 利用規約・プライバシーポリシー
- バックアップ復旧手順
- サポート運用
- 料金プラン
- 本番監視

## Phase 4 多店舗

目的: 複数店舗を管理できる構造にする。

- 店舗モデル追加
- 店舗別スタッフ・予約・決済
- 店舗別権限
- 店舗別売上
- 本部管理画面

## Phase 5 SaaS化

目的: 複数事業者へ提供できるSaaSにする。

- テナント管理
- サブスクリプション課金
- オンボーディング
- 監査ログ
- SLA
- 分析ダッシュボード
